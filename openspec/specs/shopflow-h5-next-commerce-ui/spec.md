# shopflow-h5-next-commerce-ui Specification

## Purpose
定义 `shopflow-h5-next` 第一阶段商城化 UI 的长期规格，覆盖 ShopFlow 灰蓝视觉基座、首页门面、分类浏览和商品详情购买链路。
## Requirements
### Requirement: ShopFlow H5 Next 必须提供灰蓝 ShopFlow 商城视觉基座

`shopflow-h5-next` MUST 在保留 `ShopFlow` 品牌的前提下，为第一阶段商城风格改造提供共享视觉基座。

#### Scenario: 共享 token 体现已确认的 ShopFlow 视觉方向

- **前提** 第一阶段 UI refresh 已应用
- **当** 页面消费共享 UI token
- **则** 主品牌色必须使用已确认的灰蓝方向
- **且** 常规卡片和面板圆角必须收敛在 `8px` 左右
- **且** 价格强调必须使用统一红色价格色

#### Scenario: 产品 UI 不引入 Suji 品牌

- **前提** UI refresh 借鉴 Suji 参考稿
- **当** H5 页面渲染面向用户的品牌文案
- **则** 必须继续展示 `ShopFlow`
- **且** 不得把商城改名为 `Suji`

### Requirement: 首页必须呈现第一阶段完整商城门面

`shopflow-h5-next` 首页 MUST 基于现有首页 API 数据，从迁移骨架页面转为商城首页。

#### Scenario: 首页渲染主要商城区块

- **前提** `/wx/home/index` 返回 banner、channel、coupon、groupon、新品或热销商品数据
- **当** 首页渲染
- **则** 必须展示品牌/搜索头、Banner、分类宫格和商品流
- **且** 对应数据存在时应展示优惠券和秒杀区块
- **且** 不得展示“持续迁移中”或“后续补齐”等迁移期文案

#### Scenario: 首页商品流支持第一阶段 tabs

- **前提** 首页商品列表数据可用
- **当** 用户切换首页商品流 tab
- **则** 可见商品列表必须在第一阶段支持的 feed 间切换
- **且** 空 feed 应降级为清晰空态，不得破坏导航

### Requirement: 分类页必须匹配第一阶段 ShopFlow 分类浏览布局

分类页 MUST 呈现符合已确认视觉方向的左右分栏分类浏览布局。

#### Scenario: 分类页展示侧栏和子分类内容

- **前提** 分类数据可用
- **当** 分类页渲染
- **则** 页面必须展示左侧分类侧栏
- **且** 当前选中分类必须用灰蓝选中态清晰区分
- **且** 右侧内容区必须展示当前分类上下文和子分类宫格

#### Scenario: 子分类导航保持可用

- **前提** 用户点击子分类
- **当** 页面处理点击
- **则** 必须带选中分类 id 和标题跳转到现有分类商品路由
- **且** 必须保留 `pages/items/category/index` 使用的现有路由契约

### Requirement: 商品详情页必须在采用第一阶段详情布局时保留购买行为

商品详情页 MUST 采用已确认的 ShopFlow 详情布局，同时不得回退 SKU、收藏、购物车和立即购买行为。

#### Scenario: 详情页渲染面向购买的内容层级

- **前提** 商品详情数据可用
- **当** 详情页渲染
- **则** 必须展示图集、价格、可用时的原价、商品名、简介、SKU/规格控件、数量控件和购买操作
- **且** 底部操作栏必须暴露购物车、加入购物车和立即购买操作

#### Scenario: 现有加购和立即购买流程仍然可用

- **前提** 用户已完成必要 SKU 选择
- **当** 用户点击加入购物车或立即购买
- **则** 页面必须继续调用现有购物车 API
- **且** 立即购买必须继续在跳转确认订单前写入 checkout flow context

#### Scenario: 缺失的设计稿数据安全降级

- **前提** 商品详情数据不包含真实评价、销量、地区或优惠券数据
- **当** 详情页渲染
- **则** UI 必须隐藏这些区域或使用保守标识
- **且** 不得展示伪造的评价数量或未被支持的业务承诺
