# Proposal: add-shopflow-h5-post-purchase-capabilities

## Why

当前 `shopflow-h5` 已具备商城浏览、购物车、下单、支付、订单列表与个人中心等核心交易主链路，但与 `shopflow-wx` 相比，交易后的关键用户能力仍存在明显缺口：

- 缺少售后申请、售后列表、售后详情闭环
- 缺少订单评价与商品评论列表闭环
- H5 中部分历史页面仍停留在“有入口或有接口常量，但未真正接通”的状态

这带来三类实际问题：

1. 交易闭环不完整
   - 用户完成下单后，无法在 H5 端继续顺畅处理售后与评价
   - 交易后体验弱于小程序，影响复购、客服压力和用户信任

2. 前后端能力不对齐
   - `shopflow-wx-api` 已经提供售后、订单评价、商品评论和上传等正式接口
   - H5 没有把这些现有能力接入为可用页面和完整流程

3. 历史遗留页面造成误判
   - H5 现有 `refund-list` 更接近未完成壳子
   - 订单页的“去评价”按钮已存在，但处理逻辑为空
   - 仅靠 API 常量存在，无法视为功能已完成

用户已确认本次目标不是扩展后端协议，也不是一次性补齐所有小程序运营能力，而是：

- 以 H5 前端补齐现有 `wx-api` 的交易后链路为主
- 聚焦 `P1` 范围
- 先完成售后与评论闭环
- 默认复用现有后端接口，不主动修改后端协议

## What

本次 change 将为 `shopflow-h5` 补齐以下能力：

1. 新增 H5 售后申请页
2. 新增 H5 售后列表页
3. 新增 H5 售后详情页
4. 接通 H5 订单评价发布页
5. 新增 H5 商品评论列表页
6. 在 H5 订单列表、订单详情、商品详情和个人中心中补齐相应入口与跳转
7. 接通 H5 图片上传与评论、售后表单提交流程

本次变更默认复用以下后端能力：

- `/wx/aftersale/list`
- `/wx/aftersale/detail`
- `/wx/aftersale/submit`
- `/wx/order/goods`
- `/wx/order/comment`
- `/wx/goods/comment/count`
- `/wx/goods/comment/list`
- `/wx/storage/upload`

本次变更不包含：

- 聊天、消息、动态、赏金、提现、分享等运营或社交能力
- 商家侧品牌订单管理
- H5 专属后端新协议设计
- 修改现有 `shopflow-wx-api` 的接口语义或返回结构
- 数据库结构调整

## Impact

受影响模块：

- `shopflow-h5`
- `openspec/specs/litemall-vue-compatibility`

受影响能力：

- H5 订单详情
- H5 订单列表
- H5 商品详情
- H5 用户中心
- H5 图片上传
- H5 交易后链路

关键假设：

- `shopflow-wx-api` 已有售后、评论与上传能力可直接复用
- H5 当前请求层已具备租户与用户头兼容能力，可直接承接新增页面请求
- 本次无需新增后端接口，只需在实现阶段联调确认字段消费方式

主要风险：

- H5 历史页面结构较旧，新增页面时如果继续沿用半成品页面，容易引入更多兼容负担
- 上传接口在 H5 端虽有常量，但现成上传交互与表单整合可能仍需补齐
- 如果直接复用旧 `refund-list` 页面，可能继续扩大历史债务，而不是收敛边界

预期收益：

- 让 H5 从“只有交易主链路”升级为“具备基本交易后闭环”
- 显著缩小 H5 与小程序在订单后体验上的差距
- 为后续是否继续补足迹、消息等能力建立更稳定的 H5 页面组织方式
