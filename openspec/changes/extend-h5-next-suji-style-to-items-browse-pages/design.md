# 设计：extend-h5-next-suji-style-to-items-browse-pages

## Overview

本次变更延续已确认的 `A` 方向，不再为商品列表批次重新发散新的主题分支。核心策略是：

- 保持 `ShopFlow` 当前灰蓝主题 token
- 常规卡片、列表项、内容面板统一回到 `8px` 圆角
- hero 缩小为页面识别区，不再使用大段迁移说明卡
- 搜索、分类、品牌、专题、拼团统一收敛为一套轻量商品流语言

## Shared Visual Rules

### 1. 颜色与层级

- 主色继续使用：
  - `--sf-color-brand`
  - `--sf-color-brand-light`
  - `--sf-color-brand-soft`
- 商品卡、面板、内容块继续使用白底 + 轻阴影。
- 价格强调继续使用现有价格色，不重新引入高饱和辅助色。

### 2. 圆角与阴影

- 卡片、hero、输入框、内容块、图片统一使用 `8px` 圆角基线。
- 阴影继续收敛到 `var(--sf-shadow-soft)` 和 `var(--sf-shadow-card)`。

### 3. Hero 结构

- 商品列表页 hero 只承接“当前浏览对象”身份，不再做大块营销头图。
- 搜索页以搜索栏为主，结果态使用轻 hero 承接关键词。
- 品牌 / 专题详情页允许保留图片，但首屏重量仍低于当前内容区和商品流。

### 4. 列表语言

- 搜索结果、分类结果、品牌商品流、新品、热卖、拼团页统一回到单列紧凑商品卡。
- 品牌馆列表和专题列表使用轻量内容卡，不再使用大块图卡叠压。
- 拼团信息、专题说明等特殊语义通过轻标签或轻内容卡承接，不抢商品主信息层级。

## Page Architecture

### 搜索页

- 顶部保留搜索栏。
- 未搜索时展示：
  - 热门搜索
  - 历史搜索
- 搜索后展示：
  - 轻 hero 承接关键词
  - 紧凑商品结果列表
  - 更轻的空态

### 分类结果页

- 顶部使用轻 hero 承接当前分类标题和摘要。
- 如果存在兄弟分类，使用轻胶囊列表承接跳转，不伪造排序筛选能力。
- 商品结果区使用单列紧凑商品卡，和搜索结果页统一结构。

### 品牌馆 / 品牌详情

- 品牌馆使用列表式品牌卡，不再采用重图块双列结构。
- 品牌详情页保留品牌图与简介，但压缩到轻量头部和简介卡内。
- 品牌下商品流和其他结果页统一卡片结构。

### 新品 / 热卖 / 拼团

- 三页共用同一套页面骨架：
  - 轻 hero
  - 可选轻标签摘要
  - 单列商品流
- 拼团页通过轻标签承接“几人成团 / 优惠力度”，不做高饱和运营大卡。

### 专题列表 / 专题详情

- 专题列表改成内容流式卡片，标题、摘要、封面三段式。
- 专题详情保留标题、摘要、正文卡和商品流。
- 正文块改为轻内容卡，避免首屏被大段内容压住。

## Helper Strategy

- 新增商品浏览批次展示 helper，承接：
  - 页面 eyebrow / title / description
  - 空态文案
  - 结果页、品牌页、专题页的轻量摘要文案
- 样式层使用一份商品列表批次共享基座，避免 9 个页面继续各自漂移。

## Testing Strategy

- 先为展示 helper 补测试，再落真实实现。
- 至少运行：
  - `cd shopflow-h5-next && pnpm vitest run src/features/goods/browse-display-utils.test.ts`
  - `cd shopflow-h5-next && pnpm type-check`
- 实现完成后补一次浏览域相关测试与 H5 真实页面检查：
  - `/#/pages/items/search/index`
  - `/#/pages/items/category/index?...`
  - `/#/pages/items/brand-list/index`
  - `/#/pages/items/topic-list/index`
  - `/#/pages/items/groupon/index`

## Risks and Mitigations

- 风险：搜索页和分类页过度统一后失去各自语义。
  - 处理：保留搜索栏、关键词 hero 和兄弟分类胶囊作为差异化锚点。
- 风险：品牌页和专题页如果继续保留重图块，会与已确认方向冲突。
  - 处理：图片只作为轻辅助信息，正文和商品流保持主层级。
- 风险：拼团页过度强调运营信息会破坏整体克制感。
  - 处理：团购信息只保留轻标签，不新增厚重运营区。
