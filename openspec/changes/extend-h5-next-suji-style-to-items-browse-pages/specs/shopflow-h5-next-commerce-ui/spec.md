## ADDED Requirements

### Requirement: 商品浏览批次页面必须沿用已确认的灰蓝 Suji A 方向

`shopflow-h5-next` 的搜索、分类结果、品牌、专题、新品、热卖和拼团页面 MUST 在保留 `ShopFlow` 品牌与当前灰蓝 token 的前提下，沿用已确认的轻量商品浏览方向。

#### Scenario: 主题色、hero 和圆角体系保持连续

- **前提** 本批商品列表页完成 UI 改造
- **当** 页面渲染 hero、卡片、标签、输入框和图片
- **则** 必须继续使用当前 `ShopFlow` 灰蓝主题
- **且** 常规卡片和交互块圆角必须收敛在 `8px` 左右
- **且** hero 必须保持为轻量页面识别区，不得回到重头部说明卡结构

### Requirement: 搜索和分类结果页必须在统一新风格时保留现有检索与跳转能力

搜索页和分类结果页 MUST 在统一新风格时保留当前数据流、结果展示和商品详情跳转能力。

#### Scenario: 搜索页继续支持热门、历史和结果列表

- **前提** 用户进入搜索页
- **当** 页面加载或提交关键词
- **则** 页面必须继续支持热门搜索、历史搜索、搜索结果和清空历史能力
- **且** 搜索结果中的商品卡必须继续跳转到现有商品详情页

#### Scenario: 分类结果页继续支持分类商品浏览

- **前提** 用户进入分类结果页
- **当** 页面加载分类数据和商品列表
- **则** 页面必须继续展示当前分类商品结果
- **且** 不得因为样式重构破坏现有 `categoryId` 和 `title` 路由契约

### Requirement: 品牌、专题和商品流页面必须在变轻后继续保留现有内容与商品入口

品牌馆、品牌详情、专题列表、专题详情、新品、热卖和拼团页面 MUST 在风格统一后继续保留当前内容展示和商品详情跳转能力。

#### Scenario: 品牌与专题页继续保留内容浏览入口

- **前提** 用户进入品牌馆、品牌详情、专题列表或专题详情页
- **当** 页面渲染品牌、专题和商品内容
- **则** 页面必须继续保留现有品牌 / 专题详情跳转和商品详情跳转
- **且** 专题详情页必须继续支持现有正文内容渲染

#### Scenario: 新品、热卖和拼团页继续保留现有商品流数据

- **前提** 用户进入新品、热卖或拼团页
- **当** 页面渲染商品列表和拼团信息
- **则** 页面必须继续使用当前接口字段与商品详情跳转
- **且** 拼团页不得伪造不存在的价格、人数或营销字段
