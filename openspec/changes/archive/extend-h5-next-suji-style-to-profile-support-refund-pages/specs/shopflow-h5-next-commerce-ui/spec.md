## ADDED Requirements

### Requirement: 剩余用户页必须沿用已确认的灰蓝 Suji Batch B 方向

`shopflow-h5-next` 的资料设置链路、帮助 / 服务 / 反馈页面和售后链路 MUST 在保留 `ShopFlow` 品牌与当前灰蓝 token 的前提下，沿用已确认的 Batch B 轻量用户页方向。

#### Scenario: 主题色、hero 和圆角体系保持连续

- **前提** 资料设置、帮助 / 服务 / 反馈或售后页面完成本批 UI 改造
- **当** 页面渲染 hero、卡片、输入框、按钮和状态标签
- **则** 必须继续使用当前 `ShopFlow` 灰蓝主题
- **且** 常规卡片和交互块圆角必须收敛在 `8px` 左右
- **且** hero 必须保持为轻量页面识别区，不得回到重头部结构

### Requirement: 资料设置链路必须在统一新风格时保留现有资料编辑与退出能力

资料设置页和三个资料编辑页 MUST 在统一新风格时保留当前编辑、校验和退出逻辑。

#### Scenario: 资料设置页继续提供四个资料入口

- **前提** 用户进入资料设置页
- **当** 页面渲染资料卡片列表
- **则** 用户必须仍可进入昵称、手机号和密码编辑页
- **且** 用户必须仍可切换性别并执行退出登录

#### Scenario: 资料编辑页继续保留现有提交逻辑

- **前提** 用户进入昵称、手机号或密码编辑页
- **当** 用户填写并提交内容
- **则** 页面必须继续沿用现有提交与校验逻辑
- **且** 提交成功后必须继续按当前路径返回上一页

### Requirement: 支持页和售后页必须在变轻后继续保留现有数据流与关键操作

帮助 / 服务 / 反馈页和售后链路 MUST 在风格统一后继续保留当前数据流、状态切换和关键操作入口。

#### Scenario: 帮助与服务页继续提供 FAQ 与服务入口

- **前提** 用户进入帮助中心或服务中心
- **当** 页面渲染问题列表或服务清单
- **则** 帮助中心必须继续支持 FAQ 展开和收起
- **且** 服务中心必须继续支持联系客服、帮助中心和意见反馈入口

#### Scenario: 售后链路继续支持查看与提交

- **前提** 用户进入售后列表、售后详情或申请售后页
- **当** 用户切换状态、查看详情、撤销售后或提交申请
- **则** 页面必须继续保持当前接口与路由契约
- **且** 不得因为样式重构破坏凭证上传、图片预览或撤销售后等关键能力
