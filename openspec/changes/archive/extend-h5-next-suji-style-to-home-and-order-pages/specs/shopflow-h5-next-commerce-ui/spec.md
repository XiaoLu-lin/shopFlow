## ADDED Requirements

### Requirement: 首页与订单主链路必须沿用已确认的灰蓝 Suji A 方向

`shopflow-h5-next` 的首页、订单列表、订单详情、确认订单、支付订单和支付结果页 MUST 在保留 `ShopFlow` 品牌与当前灰蓝 token 的前提下，沿用已确认的 Suji A 方向。

#### Scenario: 主题色、轻 hero 和圆角体系保持连续

- **前提** 本批首页与订单主链路页面完成 UI 改造
- **当** 页面渲染 hero、卡片、列表、金额区和底部动作区
- **则** 必须继续使用当前 `ShopFlow` 灰蓝主题
- **且** 常规卡片和交互块圆角必须收敛在 `8px` 左右
- **且** hero 必须保持为轻量页面识别区，不得回到厚重头部说明结构

### Requirement: 首页必须在统一新风格时保留现有浏览与刷新语义

首页 MUST 在统一新风格时保留当前搜索、频道、优惠和商品详情跳转能力，并避免首屏重复请求。

#### Scenario: 首页继续支持核心浏览入口

- **前提** 用户进入首页
- **当** 页面加载首页数据并渲染入口区
- **则** 页面必须继续提供搜索入口、频道入口、优惠区块、拼团入口和商品详情跳转
- **且** 不得因为样式重构破坏现有首页接口字段消费

#### Scenario: 首页首屏加载只请求一次

- **前提** 用户首次打开首页
- **当** 首屏初始化与首次 `onShow` 先后触发
- **则** 页面不得对同一份 `/home/index` 数据重复发起两次首屏请求
- **且** 后续返回首页时仍必须允许按既有语义刷新数据

### Requirement: 订单与支付页必须在统一新风格下保留现有交易操作语义

订单列表、订单详情、确认订单、支付订单和支付结果页 MUST 在统一新风格下保留现有交易状态、跳转和操作语义。

#### Scenario: 订单链路继续保留真实业务入口

- **前提** 用户进入订单列表、订单详情、确认订单或支付页
- **当** 页面渲染订单、金额、地址、优惠券和动作按钮
- **则** 页面必须继续保留现有支付、取消、确认收货、售后、地址选择、优惠券选择和提交订单能力
- **且** 不得因为样式重排导致关键入口不可达

#### Scenario: 支付结果页自动跳转不得干扰用户手动跳转

- **前提** 支付结果页处于成功态并已注册自动返回订单列表的延时跳转
- **当** 用户在倒计时结束前手动进入订单详情或订单列表
- **则** 旧定时器必须被清理
- **且** 页面不得在用户已手动跳转后再触发额外重定向
