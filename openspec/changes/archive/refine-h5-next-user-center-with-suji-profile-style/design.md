# 设计：refine-h5-next-user-center-with-suji-profile-style

## 总览

本次设计只处理 `shopflow-h5-next` 的个人中心页，目标是把当前偏交易卡片式的页面改成更贴近 `docs-suji` Profile 的 A 方案。

设计原则：

- 保留行为，重做层级
- 不伪造数据
- 尽量让页面结构本身表达“用户主页”而不是“功能列表”

## 参考稿映射

参考 `docs-suji/pages/README.md` 中 Profile 页面：

- 顶部渐变背景头部
- 中部订单状态五宫格
- 底部列表式服务入口

保留 `ShopFlow` 品牌约束：

- 文案中继续使用 `ShopFlow`
- 颜色继续使用现有灰蓝 token
- 不把产品名替换成 `Suji`

## 页面结构

### 顶部头部

- 使用渐变容器承接头像、昵称和辅助文案
- 已登录时优先展示昵称与手机号脱敏
- 未登录时展示通用欢迎文案
- 设置入口保持在头部右上

### 订单区

- 使用单卡片承接五个状态入口
- 入口顺序固定：待付款、待发货、待收货、待评价、售后
- 每个入口继续映射到现有订单列表/售后列表路由

### 服务区

- 使用列表式入口替代双列说明卡
- 每项展示标题、简短说明和右箭头
- 本次保留：
  - 购物车
  - 优惠券
  - 我的收藏
  - 收货地址
  - 退款/售后
  - 帮助中心
  - 服务中心

## 数据与兼容边界

- `fetchUserIndex()` 继续只消费订单统计
- `getSessionSnapshot()` 继续提供昵称、头像、token、手机号
- `entry-utils.ts` 继续作为跳转规则来源
- 不新增新 API，不修改页面跳转协议

## 测试策略

- 为 `dashboard-utils.ts` 补展示逻辑测试：
  - 头部文案
  - 手机号脱敏
  - 服务入口排序
- 页面修改完成后跑页面相关测试和 `type-check`
