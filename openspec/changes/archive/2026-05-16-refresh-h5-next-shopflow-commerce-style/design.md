# 设计：refresh-h5-next-shopflow-commerce-style

## 总览

本次设计采用“先统一视觉基座，再重塑三页”的策略。视觉方向来自用户提供的 `docs-suji` 参考稿，但品牌、文案和页面语义保持 `ShopFlow`。

第一阶段不做后端改造，所有页面都基于现有接口能力渐进增强。设计稿需要但接口缺失的信息必须隐藏、降级或使用明确占位，不能伪造真实业务承诺。

上游设计结论见：

- `doc/superpowers/specs/2026-05-15-shopflow-h5-next-commerce-style-design.md`

## 视觉基座

`shopflow-h5-next/src/shared/ui/tokens.css` 作为本次视觉基座入口：

- 将品牌主色从亮蓝调整为灰蓝。
- 补充价格色、浅品牌底、页面背景、边框、提示文字、阴影、圆角等 token。
- 将常规组件圆角收敛到 `8px`。
- 保持 pill 控件、搜索框、头像等使用全圆角。

`base.css` 负责同步 Vant 变量与全局基础样式，避免页面局部继续大量硬编码旧色值。

## 首页

首页采用可滚动移动商城结构：

1. 品牌搜索头：ShopFlow 品牌名 + 搜索入口。
2. Banner：使用现有 `banner` 数据。
3. 分类宫格：使用现有 `channel` 数据，最多 8 项。
4. 优惠券入口：优先使用 `couponList`，无数据时隐藏或显示低承诺空态。
5. 限时秒杀：优先使用 `grouponList`；若为空，可隐藏区块。
6. 商品 feed：用 tabs 管理 `推荐 / 新品 / 热销`，优先复用 `newGoodsList`、`hotGoodsList`，推荐可由现有可用列表兜底。

页面不得再出现迁移期说明文案。

## 分类页

分类页保留现有接口：

- `fetchCatalogList`
- `fetchCurrentCatalog`

结构调整为：

- 顶部搜索入口。
- 左侧分类栏，固定宽度，选中态灰蓝浅底。
- 右侧当前分类 Banner，优先使用当前分类图片或描述。
- 右侧 3 列子分类宫格，点击进入分类商品页。
- 空子分类时展示简洁空态。

本阶段不新增分类详情排序筛选逻辑，避免将范围扩大到 `pages/items/category`。

## 商品详情页

商品详情页保留现有业务行为：

- SKU 默认选择与切换。
- 数量加减。
- 收藏切换。
- 加入购物车。
- 立即购买并写入 checkout flow context。

视觉结构调整为：

- 顶部详情 header，包含返回、商品/评价/详情锚点和更多入口。
- 图片轮播保持现有 gallery。
- 价格、原价、折扣标签、商品名、商品 brief。
- 促销券与服务保障区作为视觉块。没有真实券数据时使用低承诺文案或隐藏。
- 商品属性、规格和数量区统一为 8px 卡片。
- 商品详情内容若存在 HTML 长图/富文本，优先渲染；否则用“详情信息整理中”的空态。
- 底部操作栏使用客服、收藏、购物车、加购、购买五段结构。

## 数据边界

- 不新增接口字段。
- 不改 API endpoint。
- 首页接入已有但未渲染的字段时，只做前端展示。
- 对销量、地区、评价数量等缺字段内容，使用隐藏或保守占位。

## 测试策略

优先补纯函数测试或可测的数据归一化逻辑：

- 首页 feed tab 数据选择。
- 首页优惠券/秒杀区块可见性。
- 商品详情折扣标签与空详情判断。

阶段验证：

- `npm run type-check`
- `npm run test`
- `npm run build:h5`
- H5 浏览器人工查看首页、分类页、详情页。
