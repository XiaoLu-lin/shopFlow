# 提案：refresh-h5-next-shopflow-commerce-style

## 背景与原因

`shopflow-h5-next` 已经具备 H5/多端迁移基线和主要业务页面骨架，但当前视觉仍保留迁移期特征：

- 首页、分类页、商品详情页仍有“迁移中”“后续补齐”等开发态文案。
- UI token 与用户提供的商城参考稿不一致，当前主色偏亮蓝，圆角和卡片体系也未收敛。
- 首页已存在 `couponList`、`grouponList`、`hotGoodsList` 等数据结构，但页面没有充分渲染。
- 分类页和商品详情页功能骨架存在，但还没有形成成熟移动商城的视觉层级和购买导向。

用户确认目标是：保留 `ShopFlow` 品牌，借鉴参考稿的灰蓝克制、白底、干净商城风格，并将首页、分类页、商品详情页作为第一阶段优先改造对象。

## 变更内容

本次 change 将在 `shopflow-h5-next` 内完成第一阶段 UI refresh：

1. 建立 ShopFlow 灰蓝商城风格 token：
   - 保留 `ShopFlow` 品牌语义。
   - 主色采用灰蓝系。
   - 组件主圆角约 `8px`。
   - 统一价格色、文字色、边框、阴影和背景色。
2. 改造首页：
   - sticky 品牌搜索头。
   - Banner、分类宫格、优惠券入口、限时秒杀、推荐/新品/热销商品流。
   - 移除迁移期提示文案。
3. 改造分类页：
   - 左侧分类栏 + 右侧 Banner + 3 列子分类宫格。
   - 保持 ShopFlow 品牌化搜索入口和底部 Tab 视觉一致。
4. 改造商品详情页：
   - 详情 header、商品图轮播、价格/优惠/保障区、详情区和底部购买栏。
   - 保留现有 SKU、收藏、加购、立即购买行为。
5. 补充必要的组件化和测试，确保现有数据请求与交易入口不回退。

本次变更不包含：

- 后端接口或数据库变更。
- 将品牌改为 Suji。
- 用户中心、购物车、结算、售后等页面的完整换肤。
- 引入新的 UI 组件库或设计依赖。

## 影响范围

受影响模块：

- `shopflow-h5-next/src/shared/ui`
- `shopflow-h5-next/src/pages/home`
- `shopflow-h5-next/src/pages/items/catalog`
- `shopflow-h5-next/src/pages/items/detail`
- 必要时涉及 `shopflow-h5-next/src/entities/*` 类型适配或纯函数测试

主要风险：

- 页面重排可能影响已有匿名浏览、分类跳转、加购、立即购买链路。
- 参考稿中的评价、销量、地区等字段并非当前接口全部具备，前端必须安全降级。
- uni-app 页面样式与浏览器 mockup 不完全等价，需要用实际 H5 构建验证。

预期收益：

- 让 `shopflow-h5-next` 的第一批核心页面具备统一、成熟的 ShopFlow 商城视觉。
- 为后续购物车、用户中心、订单等页面换肤建立 token 和组件基础。
- 消除迁移期页面观感，提升 H5 前台的可演示和可上线程度。
