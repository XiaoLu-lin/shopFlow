# 设计：extend-h5-next-suji-style-to-address-coupon-collect-pages

## Overview

本次变更采用“延续已确认个人中心 A 方向 + 统一收紧视觉重量”的策略。

核心设计原则：

- 只延展用户已确认的 `A. 轻生活感`，不重新开放主题色分支
- 继续使用既有 `ShopFlow` 灰蓝 token，不引入新的暖色主题
- 保持一点 Suji 式生活感，但整体更克制、更轻、更紧凑
- 视觉收紧优先级高于装饰表达，避免头部过重、圆角过大、模块过胖

## Scope Boundary

本次范围固定为：

- 地址列表
- 地址编辑
- 优惠券
- 收藏

明确不纳入本次：

- 帮助、服务、反馈
- 资料设置及其编辑页
- 售后相关页面
- 新的 API 或后端协议变化

## Shared Visual Rules

### 1. Color

- 统一沿用 `shopflow-h5-next/src/shared/ui/tokens.css` 中已确认的灰蓝体系：
  - `--sf-color-brand`
  - `--sf-color-brand-soft`
  - `--sf-color-brand-light`
- 不再把主题色作为页面方案变量讨论。
- 页面允许少量灰蓝渐变用于轻 hero 或强调按钮，但占比必须克制。

### 2. Radius

- 常规卡片与面板：`8px`
- 输入框与按钮：`8px`
- 票券主体：`8px`
- 仅保留胶囊标签使用更高圆角

### 3. Hero Weight

- hero 保留，但缩小为轻量页面识别区：
  - 高度降低
  - 装饰减少
  - 文案压缩
  - 与主体内容的垂直间距收紧
- hero 不得再成为页面视觉主角

### 4. Typography and Spacing

- 页面大标题、列表标题和描述字号整体收一档
- 卡片 padding 和模块 gap 统一缩小
- 页面整体以“干净、紧凑、轻”优先，不追求夸张的生活方式陈列感

## Page Architecture

### 地址列表页

- 轻量 hero 承载“收货地址”页身份
- 主体使用紧凑地址卡列表
- 每个地址块包含：
  - 姓名
  - 手机
  - 默认标记
  - 地址全文
  - 编辑入口
- 底部继续保留新增地址按钮

行为保持：

- 点击地址继续写入 `AddressId` flow context
- 选中后继续 toast + `navigateBack`
- 编辑与新增入口不改路由契约

### 地址编辑页

- 顶部 hero 仅用于区分“新增 / 编辑”
- 表单继续保留当前字段结构与顺序
- 表单块表现为轻量白卡，而不是厚重控制台表单
- 底部继续保留“删除 / 保存”双按钮布局

行为保持：

- 校验逻辑不改
- 保存逻辑不改
- 删除逻辑不改
- 暂不新增地区选择器

### 优惠券页

- 轻量 hero + tabs + 紧凑票券列表
- 票券继续保留左侧金额区和右侧内容区
- 票券视觉收紧：
  - 圆角减小
  - 间距减小
  - 字号压缩
  - 侧边强调区控制面积

行为保持：

- `未使用 / 已使用 / 已过期` 语义不改
- 继续使用当前 `status` 切换逻辑
- 继续复用 `/coupon/user` 数据流

### 收藏页

- 轻量 hero + 紧凑商品收藏列表
- 商品卡继续包含：
  - 图片
  - 商品名
  - 简述
  - 价格
  - 删除入口
- 视觉目标是“轻收藏夹”，而不是密集商品运营表

行为保持：

- 点击商品继续跳转详情
- 删除继续调用当前取消收藏逻辑
- 删除成功后继续刷新列表

## Testing Strategy

- 对页面结构主要为样式与展示调整的改动，优先补或复用现有测试，不为装饰细节强行新增复杂测试。
- 至少运行：
  - `cd shopflow-h5-next && npm run type-check`
- 实现完成后做本地 H5 视觉检查：
  - `/#/pages/user/address/index`
  - `/#/pages/user/address-edit/index?addressId=-1`
  - `/#/pages/user/coupon/index?active=0`
  - `/#/pages/user/collect/index`

## Risks and Mitigations

- 风险：收紧 hero 后页面容易过于平。
  - 处理：保留轻量灰蓝头部，但压缩高度和文案。
- 风险：8px 圆角下如果阴影仍偏重，会显得脏。
  - 处理：同步降低阴影体积和模糊范围。
- 风险：优惠券和收藏页压缩后信息可读性下降。
  - 处理：优先保主信息层级，减少非必要说明文案长度。
