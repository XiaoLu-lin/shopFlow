## ADDED Requirements

### Requirement: 用户中心延展页必须沿用已确认的灰蓝 Suji A 方向

`shopflow-h5-next` 的地址、地址编辑、优惠券和收藏页 MUST 在保留 `ShopFlow` 品牌与当前灰蓝 token 的前提下，沿用已确认的用户中心 `A. 轻生活感` 方向。

#### Scenario: 主题色与圆角体系保持连续

- **前提** 地址、地址编辑、优惠券或收藏页完成本批 UI 改造
- **当** 页面渲染用户可见卡片、输入框、按钮和票券主体
- **则** 必须继续使用当前 `ShopFlow` 灰蓝主题
- **且** 常规卡片和交互块圆角必须收敛在 `8px` 左右
- **且** 不得引入新的暖色主题作为页面主色

#### Scenario: Hero 收紧为轻量页面识别区

- **前提** 页面需要首屏头部承接身份感
- **当** 页面渲染 hero 区
- **则** hero 必须缩小为轻量识别区
- **且** 不得再使用过重、过高、过宽松的头部结构压缩主体内容

### Requirement: 地址页必须在统一新风格时保留现有地址选择与编辑行为

地址列表页和地址编辑页 MUST 在统一新风格时保留当前业务行为和路由语义。

#### Scenario: 地址列表仍支持选择、编辑与新增

- **前提** 用户进入地址列表页
- **当** 页面渲染地址列表
- **则** 用户必须仍可选择地址、进入编辑页和新增地址
- **且** 地址选择后必须继续写入现有 `AddressId` flow context

#### Scenario: 地址编辑仍保留原表单与保存删除逻辑

- **前提** 用户进入新增或编辑地址页
- **当** 用户填写并提交地址
- **则** 页面必须继续沿用现有字段结构、校验规则与保存逻辑
- **且** 编辑态下删除地址能力必须继续可用

### Requirement: 优惠券与收藏页必须在变轻后继续保留现有列表能力

优惠券页和收藏页 MUST 在风格收紧后继续保留现有数据流、状态切换和关键入口能力。

#### Scenario: 优惠券页继续支持三态切换

- **前提** 用户进入优惠券页
- **当** 用户切换未使用、已使用或已过期状态
- **则** 页面必须继续按照现有 `status` 参数加载列表
- **且** 页面必须保持当前优惠券列表接口契约

#### Scenario: 收藏页继续支持详情跳转与取消收藏

- **前提** 用户进入收藏页
- **当** 用户点击商品或执行删除
- **则** 页面必须继续支持商品详情跳转
- **且** 页面必须继续支持现有取消收藏逻辑与刷新反馈路径
