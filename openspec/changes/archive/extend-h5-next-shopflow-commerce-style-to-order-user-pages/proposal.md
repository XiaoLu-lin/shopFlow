# 提案：extend-h5-next-shopflow-commerce-style-to-order-user-pages

## 背景与原因

`shopflow-h5-next` 已完成第一阶段商城 UI refresh，首页、分类页和商品详情页已经切到灰蓝 ShopFlow 商城风格，并沉淀了共享 token 与基础视觉语言。

但交易链路与用户中心相关页面仍停留在前一批迁移基线：

- 购物车、结算、订单列表、订单详情、支付页、支付结果页和个人中心仍大量使用旧色值、旧阴影和旧圆角。
- 页面内仍保留“已接回旧站”“下一批补齐”“兼容路径”等迁移期说明文案，影响最终产品观感。
- 第一阶段已经为这些页面准备了共享 token，但目前尚未把交易链路和个人中心统一到同一套视觉系统。

用户已确认本次目标是继续参考 `docs-suji.tar.gz` 的页面组织与视觉气质，在保持 `ShopFlow` 品牌和现有接口/行为不变的前提下，推进第二阶段统一风格。

## 变更内容

本次 change 将在 `shopflow-h5-next` 内完成第二阶段 UI refresh，重点覆盖：

1. 购物车页：
   - 对齐参考稿中的单列商品卡、数量步进器、底部结算栏与购物车角标语义。
   - 保留现有勾选、全选、编辑删除、数量修改和结算跳转行为。
2. 结算页：
   - 对齐参考稿中的地址摘要、商品清单、优惠券、金额摘要与提交订单层级。
   - 保留现有 `AddressId / CartId / CouponId / UserCouponId` 下单上下文与提交逻辑。
3. 订单列表与订单详情页：
   - 对齐参考稿中的订单状态筛选、商品卡和金额/状态操作层级。
   - 保留取消、删除、确认收货、支付、售后等现有行为。
4. 支付页与支付结果页：
   - 对齐灰蓝 ShopFlow 支付面板、状态展示和结果反馈样式。
   - 保留微信内 JSAPI、H5 支付跳转和结果页路由契约。
5. 个人中心页：
   - 对齐参考稿中的用户头部、订单状态入口和快捷服务区块。
   - 保留现有订单入口、购物车入口、地址/收藏/售后等跳转行为。
6. 适度沉淀复用展示块或纯函数：
   - 仅在确实能减少重复样式或稳定逻辑测试时新增，不引入新的 UI 库或设计依赖。

本次变更不包含：

- 新增或修改后端接口。
- 改动旧 `shopflow-h5` 项目。
- 全量补齐地址、优惠券、售后、反馈、设置等所有用户页的第二阶段换肤。
- 重新设计支付协议、订单状态机或下单主流程。

## 影响范围

受影响模块：

- `shopflow-h5-next/src/pages/order`
- `shopflow-h5-next/src/pages/user`
- `shopflow-h5-next/src/features/cart`
- `shopflow-h5-next/src/features/order`
- 视需要涉及 `shopflow-h5-next/src/shared/ui`

主要风险：

- 交易链路页面样式重排可能误伤现有勾选、下单、支付、订单操作和跳转行为。
- 参考稿中的角标、状态、销量或图标布局需要适配现有接口字段，前端必须做保守降级，不能伪造数据。
- 个人中心和订单列表都承载多个跳转入口，若结构改动不谨慎，容易引入路由回归。

预期收益：

- 让 `shopflow-h5-next` 从“商城首页已新、交易页仍旧”变为一套连续的 ShopFlow 购物体验。
- 消除迁移期提示文案，让 H5 Next 更适合作为对外演示和后续上线基线。
- 为后续地址、优惠券、售后等剩余页面的换肤提供交易页/个人中心基座。
