# ShopFlow H5 Next 商品列表批次 Suji A 方向设计稿

## 背景

`shopflow-h5-next` 的首页、分类入口、商品详情页，以及订单和个人中心相关页面已经统一到灰蓝 ShopFlow / Suji 方向，但剩余商品浏览类页面仍停留在较早的迁移期样式中。

用户已在本地 HTML 预览中明确确认，本批商品列表页继续采用 `A` 方向，并要求延续之前的协作方式：先出本地预览，再确认，再改真实页面。

## 已确认结论

- 本次范围固定为：
  - `search`
  - `category`
  - `brand-list`
  - `brand`
  - `new`
  - `hot`
  - `topic-list`
  - `topic`
  - `groupon`
- 视觉方向固定为已确认的 `A` 方向。
- 对外品牌仍保持 `ShopFlow`，不引入 `Suji` 作为产品名。
- 继续使用灰蓝主题、`8px` 圆角、小 hero、紧凑间距。
- 不修改现有接口、路由契约和商品详情跳转行为。

## 目标

把这一批页面从“早期迁移骨架页”统一成一套连续的商品浏览域：

- 搜索页更像“搜索发现 + 搜索结果”
- 分类结果页更像“轻量结果列表”
- 品牌页和专题页更像“内容 + 商品流”
- 新品、热卖、拼团页更像“同一家店里的不同商品 feed”

## 结构方案

### 搜索页

- 顶部保留搜索栏。
- 未搜索时展示热门搜索和历史搜索。
- 搜索后使用轻 hero 承接关键词，并直接进入紧凑商品结果列表。
- 空态继续保留，但文案改成正式产品语气。

### 分类结果页

- 顶部使用轻 hero 承接当前分类名与摘要。
- 如果有兄弟分类，使用轻胶囊列表承接当前上下文。
- 商品结果区统一使用单列商品卡，和搜索结果页语言一致。

### 品牌馆 / 品牌详情

- 品牌馆从双列大图卡改为列表式品牌卡。
- 品牌详情页保留品牌图和简介，但压缩为轻头部和简介卡。
- 品牌下商品列表与其他商品流页统一。

### 专题列表 / 专题详情

- 专题列表改成标题、摘要、封面三段式内容卡。
- 专题详情页使用轻 hero + 正文卡 + 商品流。
- 保留正文渲染能力，但视觉重量低于商品流主区。

### 新品 / 热卖 / 拼团

- 三页共用同一套页面骨架。
- 用轻 hero 和轻标签区分语义，不再各自长出一套不同的列表结构。
- 拼团信息通过轻标签承接，不做高饱和运营块。

## 数据与降级

- 不伪造品牌故事、专题销量、拼团热度等不存在的数据。
- 如果分类摘要、品牌描述或专题副标题为空，使用保守、正式的降级文案。
- 真实存在的商品价格、原价、拼团人数和拼团优惠字段继续展示。

## 测试与验证

- 优先为商品浏览批次展示 helper 补测试。
- 页面改造完成后运行：
  - `cd shopflow-h5-next && pnpm vitest run src/features/goods/browse-display-utils.test.ts`
  - `cd shopflow-h5-next && pnpm type-check`
- 再对搜索、分类、品牌、专题和拼团页做 H5 本地视觉检查。
