# ShopFlow H5 Next 商城风格设计稿

## 背景

`docs-suji.tar.gz` 提供了一套移动商城视觉参考，核心气质是灰蓝克制、白底、简约干净、逛不累。用户确认本项目不改名为 Suji，仍保留 `ShopFlow` 品牌，只吸收这套视觉语言和页面组织方式。

本次为 `shopflow-h5-next` 第一阶段风格改造，范围限定为：

- 首页
- 分类页
- 商品详情页
- 支撑这三页的基础视觉 token 与可复用展示组件

不在本阶段改后端接口、不回溯已下线旧 H5 工程、不处理结算/用户中心/售后等后续页面。

## 已确认结论

- 品牌名称继续使用 `ShopFlow`。
- 主视觉方向采用灰蓝克制、白底、轻阴影、清晰留白。
- 组件主圆角收敛到约 `8px`，不沿用参考稿里偏大的 `12px+` 圆角。
- 第一阶段采用偏激进改版：页面结构和交互尽量靠近参考稿，而不只是换色。
- 首页、分类页、商品详情页优先形成统一视觉效果，其他页面后续分批跟进。

## 视觉方向

### 色彩

主色使用参考稿的灰蓝色系，但以 `ShopFlow` 命名落入项目 token：

- Brand: `#4A6FA5`
- Brand light: `#6D8CB8`
- Brand soft: `#E5EDF6`
- Page background: `#F8F9FB`
- Card background: `#FFFFFF`
- Border/divider: `#E8ECF1` / `#F0F2F5`
- Primary text: `#2C3E50`
- Secondary text: `#7B8BA3`
- Price/action red: `#E8553D`

### 形状与表面

- 常规卡片、Banner、商品图、面板默认使用 `8px` 圆角。
- 搜索框、标签、复选框等 pill/圆形控件保留全圆角。
- 大面积页面外壳或浏览器 mockup 不进入产品实现，不作为产品圆角依据。
- 阴影控制在轻量层级，优先用边框和背景区分层级。

### 字体层级

- 中文字体保持现有移动端安全字体栈，以 `PingFang SC` 优先。
- 商品标题、价格、模块标题建立明确层级。
- 价格统一使用红色并加粗，原价使用浅色删除线。

## 页面方向

### 首页

首页从当前“迁移基线页面”调整为真实商城首页：

- 顶部 sticky ShopFlow header，左侧品牌，右侧搜索入口。
- Banner 轮播采用灰蓝品牌基调、8px 圆角。
- 分类 4x2 宫格，点击进入对应分类或专题。
- 优惠券入口使用浅灰蓝到白色渐变，突出领取动作。
- 限时秒杀横向滚动商品卡，展示现有 `grouponList` 或可用促销数据。
- Feed tabs 使用 `推荐 / 新品 / 热销`，下方商品区使用双列瀑布流或接近瀑布流的双列卡片。
- 移除“迁移中”“后续补齐”等开发态文案。

### 分类页

分类页独立呈现，不再和详情页混合：

- 顶部保留 ShopFlow 品牌化搜索入口。
- 左侧固定分类栏，选中态使用灰蓝浅底。
- 右侧显示当前分类 Banner 与 3 列子分类宫格。
- 下方可以展示热卖分类或当前分类推荐入口，但不在本阶段引入新接口。
- 底部 Tab 与整体品牌色一致。

### 商品详情页

商品详情页独立呈现，形成完整购买页面：

- 顶部详情 header，包含返回、`商品 / 评价 / 详情` 锚点、更多入口。
- 商品图轮播使用 1:1 区域，8px 圆角或贴近移动端详情页的轻边框。
- 价格区展示现价、原价、折扣标签。
- 标题、副标题、已售/免运费/地区等元信息建立层级；缺字段时使用可控占位或隐藏，不伪造后端数据。
- 促销券和服务保障区作为独立视觉块。
- 详情图片区与评价区按现有数据能力渐进实现；无真实评价接口时允许使用结构占位，但不能展示误导性真实评价数量。
- 底部操作栏包含客服、收藏、购物车、加购、购买，并使用灰蓝/红橙按钮区分主次动作。

## 组件方向

优先沉淀小而明确的复用组件或样式块：

- `ShopHeader` 或页面内同等结构：品牌 + 搜索入口。
- `ProductCard`：支持双列卡与单列卡两种展示。
- `CouponBanner`：首页入口和详情页券块可复用视觉语言。
- `SectionHeader`：标题、辅助操作、倒计时。
- `DetailActionBar`：商品详情底部操作区。

如果现有工程更适合先用页面内 scoped style 实现，也可以先保持页面内实现，但 token 必须统一，避免每页硬编码一套颜色。

## 数据与兼容性

- 继续复用 `entities/home/api.ts`、`entities/catalog/api.ts`、`entities/goods/api.ts`、`entities/cart/api.ts`、`entities/user/api.ts`。
- 本阶段不要求后端新增字段。
- 已有但未渲染的首页数据，如 `couponList`、`grouponList`、`hotGoodsList`，应优先接入。
- 对设计稿需要但接口缺失的数据，如真实评价数量、已售数量、发货地，前端应做安全降级，不展示虚假强承诺。

## 风险

- 当前项目是 uni-app + Vue 3，不是纯 H5 Vue SPA，部分 CSS/DOM 能力需要兼容 uni 语法。
- 参考稿里的局部交互动效需要按 uni-app 约束落地，不应引入新依赖。
- 如果一次性重写三个页面过大，优先保证 token 和首页完成，再推进分类与详情。

## 验证目标

- `npm run type-check`
- `npm run test`
- `npm run build:h5`
- 浏览器/移动视口人工确认首页、分类页、详情页首屏和关键交互。
