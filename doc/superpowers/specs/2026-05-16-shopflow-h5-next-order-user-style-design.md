# ShopFlow H5 Next 第二阶段交易链路与个人中心风格设计稿

## 背景

`shopflow-h5-next` 第一阶段已经完成首页、分类页和商品详情页的 ShopFlow 商城化改造，并沉淀了灰蓝 token、基础卡片体系和部分页面组织方式。

用户本次确认继续参考 `/Users/lhl/Desktop/code/docs-suji.tar.gz`，把第二阶段范围收敛到交易链路和个人中心页面，目标是让 H5 Next 从“只有逛的页面像商城”升级为“逛、买、支付、查订单、看个人中心都像同一套商城”。

## 已确认结论

- 品牌继续使用 `ShopFlow`，不引入面向用户的 `Suji` 命名。
- 第二阶段继续沿用第一阶段已确认的灰蓝主色、白底和轻阴影。
- 以 `docs-suji/pages/README.md` 中的 Cart、Profile 与交易型页面层级作为布局参考，但不生搬用户 ID、销量、角标等设计稿专属数据。
- 不改现有 API、路由契约、checkout flow context 和支付流程。
- 交易页与个人中心页中所有迁移期提示文案都应被移除或替换成正式产品文案。

## 页面范围

本次范围固定为：

- 购物车页 `pages/order/cart`
- 结算页 `pages/order/checkout`
- 订单列表页 `pages/user/order-list`
- 订单详情页 `pages/order/detail`
- 支付页 `pages/order/payment`
- 支付结果页 `pages/order/payment-status`
- 个人中心页 `pages/user`

本次不纳入：

- 地址列表与地址编辑
- 收藏、优惠券、帮助、服务、反馈、售后等子页面的完整换肤
- 搜索浮层与分类详情页第二阶段扩展
- 后端接口与支付协议变更

## 视觉方向

### 统一基线

- 页面背景继续使用 `#F8F9FB`
- 卡片与面板保持白底
- 主品牌色继续使用 `#4A6FA5`
- 价格与支付强调继续使用 `#E8553D`
- 常规卡片、商品图、按钮默认仍以约 `8px` 为主圆角
- 头像、搜索框、状态 chip 和勾选控件可保留全圆角

### 交易链路层级

- 交易页要比首页更强调“状态”和“金额”
- 商品信息区优先突出图、名称、规格、数量和价格
- 支付/下单/确认等主操作使用品牌色或价格色按钮，次操作使用浅底或描边
- 金额摘要与状态摘要必须比解释性文案更醒目

### 个人中心层级

- 顶部头部区需要明显区别于普通列表页，形成“账户主页”感
- 订单状态入口应一眼可扫读，优先看数量和语义
- 快捷服务入口改为更正式、更统一的服务卡片，不再像迁移待办列表

## 页面结构建议

### 购物车

- 顶部标题区
- 单列购物车商品卡
- 编辑态数量与删除切换
- 底部固定合计与结算栏

### 结算

- 地址摘要卡
- 商品清单卡
- 优惠券选择区
- 备注输入区
- 金额摘要与提交栏

### 订单列表

- 标题区
- 状态 chips 滚动条
- 订单卡列表
- 空态

### 订单详情

- 状态摘要卡
- 商品卡
- 收货信息卡
- 订单信息卡
- 金额与物流卡
- 底部操作区

### 支付与支付结果

- 支付信息摘要
- 支付方式卡片
- 支付结果状态卡
- 回到订单列表入口

### 个人中心

- 顶部用户头部
- 订单状态入口区
- 快捷服务区

## 数据与降级

- 若参考稿中的角标数量、用户 ID、标签文案没有真实字段支持，则隐藏或替换为保守文案。
- 支付页中支付宝入口可继续保留“待补齐”语义，但视觉上必须明确它不是当前主链路。
- 个人中心若无更多用户画像信息，不新增伪造个人资料字段。

## 风险

- 交易链路页面都带真实操作，样式重构时最容易引入点击区域、滚动区域或按钮层级问题。
- 支付结果页有自动跳转逻辑，页面重构时不能破坏原有时序。
- 个人中心入口多，若快捷入口分组不谨慎，容易把原有跳转埋没。

- `cd shopflow-h5-next && npm run type-check`

- `cd shopflow-h5-next && npm run build:h5`
- H5 视口下人工检查第二阶段页面首屏和核心交互。
