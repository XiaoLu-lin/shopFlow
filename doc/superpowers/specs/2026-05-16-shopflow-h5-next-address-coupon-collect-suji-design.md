# ShopFlow H5 Next 地址 / 优惠券 / 收藏 Suji A 方案设计稿

## 背景

`shopflow-h5-next` 的个人中心首页已经完成 `A. 原版 Suji 感` 方向确认，但其下游用户页目前仍停留在上一轮较重、较工具化的交易页风格中。

用户已确认本批次优先处理以下页面，并继续采用“先出本地预览，再选 UI 方向，再改真实页面”的协作方式：

- `src/pages/user/address/index.vue`
- `src/pages/user/address-edit/index.vue`
- `src/pages/user/coupon/index.vue`
- `src/pages/user/collect/index.vue`

用户在本地预览中已明确选择：

- 页面方向：`A. 轻生活感`
- 气质要求：接近 `docs-suji` 参考稿，但不改 `ShopFlow` 品牌和主题色
- 视觉修正：统一收紧圆角，不要过大的圆角和过重的 hero

## 已确认结论

- 视觉方向固定为 `A. 轻生活感`。
- 对外品牌继续保持 `ShopFlow`，不引入面向用户的 `Suji` 命名。
- 主题色不再作为变量讨论，统一沿用当前项目已确认的灰蓝色 token：
  - `--sf-color-brand: 74 111 165`
  - `--sf-color-brand-soft: 229 237 246`
  - `--sf-color-brand-light: 109 140 184`
- 本批页面统一采用更克制的圆角体系，常规卡片、按钮、输入框和票券主体以 `8px` 左右为基线。
- hero 头部保留，但只作为轻量首屏锚点，不得再做成大面积、强存在感的重头部。
- 不修改现有接口、路由、登录态、选择语义和关键跳转行为。

## 目标

把地址、优惠券、收藏这一组用户页统一到和个人中心 A 方案连续的用户视觉体系中：

- 更像完整用户侧页面，而不是单纯的交易表单或列表壳子
- 保留一点生活化、日常感
- 整体更轻、更干净、更克制
- 避免圆角过大、模块过胖、头部过重的问题

## 范围

本次只覆盖：

- 地址列表页
- 地址编辑页
- 优惠券列表页
- 收藏列表页

本次不包含：

- 帮助、服务、反馈
- 资料设置页及各编辑页
- 售后相关页面
- 后端接口、数据结构和请求协议调整

## 视觉系统要求

### 1. 颜色

- 继续使用现有 `ShopFlow` 灰蓝体系，不再引入暖杏、蜜桃、奶油等偏色方案。
- 背景维持浅雾白与灰蓝渐层的轻对比关系。
- 强调按钮、票券侧边、轻标签等重点部位可使用灰蓝渐变，但占比必须克制。

### 2. 圆角

- 常规卡片：`8px`
- 内层信息块：`8px`
- 输入框、按钮：`8px`
- 只在胶囊标签等弱控件保留更高圆角

### 3. hero 头部

- 每页可保留小型 hero，但必须缩小：
  - 更矮
  - 更轻
  - 更少文字
  - 更少装饰
- hero 只负责传达页面身份和一点点用户页情绪，不承担大面积首屏占比。

### 4. 字号与间距

- 相比当前预览早期版本，整体字号与留白统一收一档。
- 信息主标题、列表标题、标签和描述都应偏轻量，避免页面显得“软大厚”。

## 页面结构方案

### 1. 地址列表页

- 顶部使用轻量 hero 说明“收货地址”。
- 主体改为紧凑的地址卡片列表。
- 每张卡片保留：
  - 收货人
  - 手机号
  - 默认标记
  - 地址正文
  - 编辑入口
- “新增地址”按钮继续固定在底部，但尺寸、圆角和阴影统一收紧。

关键要求：

- 继续保留旧站 `AddressId` 选中语义。
- 点击地址仍写入 flow context 并返回上页。
- 页面气质应更像“我的地址簿”，而不是结算页附属弹层。

### 2. 地址编辑页

- 顶部保留轻量 hero，仅用于区分“新增地址 / 编辑地址”。
- 表单分组继续保留，但表现要更轻：
  - 收货人
  - 手机号
  - 省市区
  - 详细地址
  - 地区编码 / 邮编
  - 默认地址开关
- 删除 / 保存按钮继续保留双按钮结构，但按钮视觉统一收紧。

关键要求：

- 不改变现有表单校验逻辑和提交逻辑。
- 不新增复杂地区选择器，本轮仍保留文本输入模式。
- 页面视觉上不能再像重表单控制台。

### 3. 优惠券页

- 顶部保留轻量 hero，与地址页保持同一头部语气。
- tabs 保留“未使用 / 已使用 / 已过期”语义。
- 列表继续使用票券感，但视觉降噪：
  - 票券主体更紧凑
  - 圆角缩小
  - 灰蓝体系统一
  - 避免过多装饰性切口和夸张色块

关键要求：

- 继续复用已接通的 `/coupon/user` 数据流。
- 不修改状态切换逻辑。
- 气质更像用户票夹，而不是营销活动 banner。

### 4. 收藏页

- 顶部使用轻量 hero，与个人中心和地址页保持一致。
- 列表仍按商品卡展示，但更像“我的收藏清单”：
  - 图片
  - 商品名
  - 简述
  - 价格
  - 删除入口
- 卡片结构要轻，避免厚重阴影与过大圆角。

关键要求：

- 保留商品详情跳转与取消收藏行为。
- 不破坏当前删除后刷新列表的反馈路径。
- 页面应呈现“日常收藏夹”感，而不是纯商品运营列表。

## 行为与数据约束

- 地址、优惠券、收藏的现有 API 路径与请求参数不做修改。
- 不伪造用户身份、等级、积分、券面规则等设计稿没有的数据。
- 继续复用现有：
  - `src/entities/user/api.ts`
  - `src/shared/compat/session-adapter.ts`
  - `src/shared/ui/tokens.css`

## 建议 change 名称

`extend-h5-next-suji-style-to-address-coupon-collect-pages`

该名称用于表达：

- 范围属于 `shopflow-h5-next`
- 是已确认 Suji A 方向的延展批次
- 页面范围只覆盖地址、优惠券、收藏

## 测试与验证

- 页面展示逻辑改动优先补对应测试或现有测试扩展。
- 至少运行：
  - `cd shopflow-h5-next && npm run type-check`
- 实现完成后在本地 H5 页面检查：
  - `/#/pages/user/address/index`
  - `/#/pages/user/address-edit/index?addressId=-1`
  - `/#/pages/user/coupon/index?active=0`
  - `/#/pages/user/collect/index`

## Gate 1 结论

- 当前理解：继续沿个人中心 A 方案，把地址、地址编辑、优惠券、收藏统一到同一套 Suji 化用户页风格。
- 范围：只做这四页，不扩到资料设置、帮助服务或售后。
- 候选方案：本轮预览曾比较 `A / B / C`，用户最终明确选择 `A`。
- 推荐方案：`A. 轻生活感`，并在用户反馈后进一步收紧为 `8px` 圆角、小 hero、轻字号、轻间距版本。
- 拟定 change 名称：`extend-h5-next-suji-style-to-address-coupon-collect-pages`
