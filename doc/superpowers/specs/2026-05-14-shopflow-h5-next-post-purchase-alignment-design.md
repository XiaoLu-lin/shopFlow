# shopflow-h5-next 交易后链路对齐设计

## 背景

`shopflow-h5-next` 当前已经是新的 `uni-app` 多端主工程，主交易链路基本可用，售后链路也已有一版实现。但与 `shopflow-wx` 对照后可以确认：

- 售后不是“完全没有”，而是“已接了一版但还没完全对齐”
- 评价闭环则仍明显缺失

这意味着当前最合理的方向不是重做整段交易后链路，而是：

1. 基于现有 `shopflow-h5-next` 售后实现继续收口
2. 把 `shopflow-wx` 已存在的评价链路补到 `shopflow-h5-next`
3. 同时把售后规则和入口语义拉齐到小程序

## 已确认范围

本次只覆盖 `shopflow-h5-next` 的交易后链路，不扩展到其他业务域。

纳入范围：

- 订单评价发布页
- 商品评论列表页
- 订单列表 / 订单详情“去评价”入口
- 商品详情评论入口与评论摘要
- 售后申请校验规则对齐
- 售后列表状态分组对齐
- 订单详情退款 / 售后入口语义对齐

不纳入范围：

- 聊天
- 消息
- 动态
- 足迹
- 赏金
- 商家侧售后审核

## 现状判断

### `shopflow-h5-next` 已有部分

- 已有售后申请页、售后列表页、售后详情页
- 已有 `aftersale/list`、`aftersale/detail`、`aftersale/submit`、`storage/upload` API 封装
- 已有订单详情中的“申请售后”入口
- 已有售后相关 e2e 冒烟用例

### `shopflow-h5-next` 当前缺口

- 没有订单评价发布页
- 没有商品评论列表页
- 订单列表没有“去评价”入口
- 订单详情没有“去评价”入口
- 商品详情没有评论入口与评论摘要
- 售后申请页未完整对齐 `shopflow-wx` 的必填与凭证规则
- 售后列表的 tab 语义与 `shopflow-wx` 不一致

## 推荐方案

推荐采用“增量对齐”方案：

1. 继续保留 `shopflow-h5-next` 当前售后页面与 API 封装
2. 在 `pages` 层新增评价页面与评论列表页面
3. 在订单列表、订单详情、商品详情中补入口
4. 对售后申请校验和售后列表状态文案做定向修正
5. 不修改后端协议，不引入新的服务边界

### 为什么不建议直接复用旧 `shopflow-h5` 页面

- 技术栈已经切到 `uni-app + Vue 3 + TypeScript`
- 旧 H5 页面结构与当前 `shopflow-h5-next` 页面组织方式不一致
- 直接迁旧页面会把历史页面耦合再次带入新工程

因此更合适的方式是：

- 复用旧 H5 和 `shopflow-wx` 的业务语义
- 不复用其页面实现

## 页面与数据流设计

### 订单评价页

入口：

- 订单列表
- 订单详情

数据流：

1. 根据 `goodsId` 调用 `order/goods`
2. 展示待评价商品信息
3. 用户填写星级、评价内容、可选图片
4. 图片通过 `storage/upload` 获取 URL
5. 通过 `order/comment` 提交
6. 成功后返回上一页面并刷新评价态

### 商品评论列表页

入口：

- 商品详情评论模块

数据流：

1. `goods/comment/count` 拉评论总数与有图数
2. `goods/comment/list` 拉分页列表
3. 支持全部 / 有图切换
4. 支持下拉或触底分页

### 售后申请规则对齐

对齐点：

- 售后类型必选
- 退款原因必填
- 退款说明必填
- 非“未收货退款”必须上传凭证

保持不变：

- 继续复用当前 `/wx/aftersale/submit`
- 继续复用当前图片上传 API

### 售后列表语义对齐

目标不是重做售后列表，而是把 tab 文案和分组认知调整为与 `shopflow-wx` 一致，避免 H5 与小程序给用户两套不同状态语言。

## 风险与约束

主要风险：

- 当前订单接口在 `shopflow-h5-next` 中对 `comment` / `refund` / `aftersale` 的消费可能还不完整，实现时需要先核清楚字段口径
- 商品详情增加评论入口时，要避免破坏购买主路径布局
- 售后状态对齐如果只改文案不改映射，可能造成“看起来一样但筛选结果不一样”

约束：

- 不新增后端接口
- 不改数据库
- 不把范围扩展到非交易后模块

## 验证思路

至少验证：

1. 订单列表进入评价页
2. 订单详情进入评价页
3. 评价可提交
4. 商品详情进入评论列表
5. 评论列表支持全部 / 有图切换
6. 售后申请必填校验生效
7. 非未收货退款要求凭证
8. 售后列表状态切换可用
9. `shopflow-h5-next` H5 构建通过
