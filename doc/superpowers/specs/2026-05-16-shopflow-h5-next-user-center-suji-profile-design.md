# ShopFlow H5 Next 个人中心 Suji Profile A 方案设计稿

## 背景

`shopflow-h5-next` 的个人中心页已经完成第二阶段基础换肤，但当前实现更偏交易页卡片拼接，和用户确认想要的 `docs-suji` Profile 页面气质仍有差距。

用户已在本地视觉预览中明确选择 `A. 原版 Suji 感`，并确认后续页面也采用“先出本地网站预览，再选 UI 方向，再改真实页面”的协作方式。

## 已确认结论

- 本次范围只包含 `shopflow-h5-next/src/pages/user/index.vue`。
- 视觉方向固定为 `A. 原版 Suji 感`。
- 对外品牌仍保持 `ShopFlow`，不引入 `Suji` 作为产品名。
- 保留现有接口、登录态、路由与页面入口行为。
- 订单、购物车、收藏、地址、优惠券、售后、帮助、服务、设置入口都必须继续可用。

## 目标

把个人中心页从“说明卡 + 宫格卡片”改成更接近 `docs-suji/pages/README.md` 中 Profile 的结构：

- 顶部渐变用户头部
- 中部“我的订单”五宫格
- 底部列表式服务入口

整体气质要更像一个完整的用户主页，而不是若干功能卡片的堆叠。

## 结构方案

### 1. 顶部用户头部

- 使用灰蓝渐变大头部作为首屏锚点。
- 左侧展示头像，右侧展示昵称与一行辅助信息。
- 辅助信息优先使用已登录用户的手机号脱敏文案；若没有手机号，则回退成通用会员说明。
- 右上保留“设置”按钮，继续跳转现有设置页。

### 2. 我的订单区

- 使用白底卡片承接五个订单状态入口。
- 入口顺序保持：待付款、待发货、待收货、待评价、售后。
- 每个入口展示状态标签与数量/查看文案。
- 卡片头部保留“全部订单”入口，继续跳到现有订单列表页。

### 3. 服务入口区

- 不再使用双列说明型大卡片。
- 改成更贴近参考稿的列表式服务入口。
- 每一行只承载一个服务入口，左侧为标题与一句简短说明，右侧为箭头。
- 建议顺序：
  - 购物车
  - 优惠券
  - 我的收藏
  - 收货地址
  - 退款/售后
  - 帮助中心
  - 服务中心

## 数据与降级

- 不伪造用户 ID、等级、积分、勋章等参考稿中不存在的数据。
- 头像、昵称、token、手机号均继续来自现有 session snapshot。
- 若手机号不存在，则头部辅助文案回退为通用会员说明。
- 订单状态数量继续使用 `/user/index` 返回的现有统计字段。

## 测试与验证

- 优先为个人中心展示辅助逻辑补测试：
  - 头部文案与手机号脱敏
  - 服务入口分组/排序
- 页面改造完成后运行：
  - `cd shopflow-h5-next && npm run test -- src/pages/user/dashboard-utils.test.ts src/pages/user/entry-utils.test.ts`
  - `cd shopflow-h5-next && npm run type-check`
- 最后在本地 H5 页面 `/#/pages/user/index` 人工检查首屏视觉和入口点击。
