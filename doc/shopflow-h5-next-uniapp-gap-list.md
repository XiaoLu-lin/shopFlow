# `shopflow-h5-next` uni-app 迁移缺口清单

更新日期：2026-05-13

## 已完成主链路

- 浏览域：首页、分类、搜索、商品详情、热卖、新品、拼团、品牌、专题
- 鉴权域：登录、注册、找回密码
- 交易域：购物车、结算、支付、支付结果、订单详情
- 用户域：个人中心、订单列表、地址、收藏、优惠券、退款/售后、售后详情、售后申请、资料设置、帮助、服务、反馈

## 当前仍有缺口

### 1. 资料编辑链路

- 手机号编辑当前只做了兼容存储写回，未真正走后端资料更新接口
- 头像编辑尚未迁移
- 昵称/手机号/密码页的成功后联动还可继续加强

### 2. 售后链路细节

- 售后申请成功后，仅通过页面返回和回刷更新，未做更细的跨页状态同步
- 售后图片上传仍沿用 H5 文件输入方式，后续需要补小程序端选择图片体验
- 售后列表和订单列表之间的状态联动还可以继续压实

### 3. 用户中心质量项

- 优惠券、收藏、帮助、服务等页面已可用，但空态/异常态文案还可统一
- 个人中心信息密度和视觉层级还能再精修一轮
- 部分页面的返回路径与 redirect 语义还缺一轮系统性冒烟

### 4. 多端专项缺口

- 当前主要按 H5 路径持续验证，小程序端需要再做一轮集中冒烟
- 文件上传、图片预览、支付回跳等能力需要补微信小程序侧手工验证
- uView Plus 的 Sass 警告仍存在，虽不阻塞运行，但后续可安排治理

### 5. OpenSpec / 文档治理

- `openspec/changes/rebuild-shopflow-h5-next-as-uniapp-multiplatform/tasks.md` 仍停留在前置设计任务，尚未同步 apply 实际进展
- 需要补一份实现进度与 review 摘要，方便后续归档

## 建议的下一批优先级

1. 手机号编辑改为真实后端资料更新
2. 小程序端集中冒烟：首页、分类、详情、购物车、登录、个人中心
3. 统一用户域空态、异常态、提交成功态文案
4. 同步 OpenSpec tasks / review / verify 证据
