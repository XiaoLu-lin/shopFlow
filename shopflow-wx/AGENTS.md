# shopflow-wx 目录说明

> 适用范围：`shopflow-wx` 及其子目录，除非被更深层 `AGENTS.md` 覆盖。

## 1. 本目录职责

- 目录名称：`shopflow-wx`
- 主要职责：提供微信小程序前端，承载页面、组件、配置、工具方法和用户侧交互流程。
- 不负责内容：不承担后端业务规则，不替代 `shopflow-wx-api`；不把跨页面公共逻辑长期复制到多个页面中。

## 2. 复用与依赖边界

- 优先复用入口：`utils`、`lib`、`custom-tab-bar` 和既有页面通用逻辑。
- 优先复用入口：已有授权、请求、支付、分享、列表与详情类页面模式。
- 对外依赖边界：通过统一请求与工具封装调用后端，避免在页面文件里散落重复网络与状态处理代码。
- 对外暴露边界：为用户提供稳定、可预期的小程序浏览、下单、支付、个人中心和互动体验。

## 3. 编码约束

- 页面交互和接口字段改动前，先确认是否影响已有页面、缓存、分享链路和授权流程。
- 购物车、下单、支付、登录、消息、动态等主路径优先保证稳定，不做未经设计的行为漂移。
- 优先抽公共逻辑到 `utils` 或可复用组件，避免页面间复制粘贴。
- 兼容微信小程序运行环境，避免引入与当前工程模式不一致的实现。

## 4. 测试与验证

- 当前目录修改后，优先验证：页面渲染、授权登录、列表与详情流转、下单支付、分享和异常提示。
- 高风险点：登录态同步、购物车状态、订单提交、支付回调后的前端表现、图片与资源加载、聊天与消息页面。
- 变更后需要同步检查：对应后端接口、页面配置、静态资源、分包与发布配置。

## 5. OpenSpec 衔接

- 涉及当前目录的功能新增、行为变更、主流程调整时，默认先走 OpenSpec。
- `apply` 完成后自动进入 `review`。
- `review` 完成后停止等待用户确认，再决定是否 `archive`。
- 如果归档结论只影响当前目录，优先沉淀到当前 `AGENTS.md`。
